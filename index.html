Java Swing GUI OOP Game Logic
Personal Project

2-Player Interactive Chess

Java — Full Rule Enforcement with Swing GUI

I built this mostly to see if I could. A fully working chess game in Java, no libraries, just figuring it out from scratch. Chess sounds simple until you actually try to model the insanely complex logic of it in code.

The game has a full GUI built with Java Swing. Two players take turns clicking to move pieces on a rendered board. I implemented the full ruleset: every piece has its own legal move generation, and the game handles check, checkmate, castling, en passant, and pawn promotion. Getting check detection right was the hardest part. Every candidate move has to be simulated first to make sure it doesn't leave your own king exposed, which adds a layer of complexity to basically every move in the game.

The bigger challenge was keeping the codebase organized. Each piece is its own class, the board state is managed separately from the rendering, and the move validation logic sits in between. It got complex fast, and a lot of what I learned from this project was less about chess and more about structuring code that actually scales.

Full rule enforcement: check, checkmate, castling, en passant, and pawn promotion
Graphical board built with Java Swing, fully interactive click-based interface
Each piece is its own class with individual move logic, board state managed centrally
Move validation simulates each candidate move before allowing it

Personal Project

Super Mario Bros. Platform Tiler

Python — Built from Scratch, No Game Libraries

I built a Super Mario Bros.-inspired platformer in Python to see how games actually work under the hood. Things like the main loop, how input gets processed, how everything stays in sync each frame.

The levels are tile-based, so the world is basically a grid of platforms, gaps, and walls that the player moves through. I wrote the movement, gravity, and collision detection myself. Getting the collision to feel right took more iteration than I expected. There are a lot of edge cases when a character is landing on a tile at an angle or clipping a corner.

It was a good first look at what it means to write a program that runs continuously rather than just executing top to bottom. Different way of thinking about code structure.

Tile-based level system with grid-defined world layout
Basic player physics: gravity, movement, and jumping
Tile collision detection so the player interacts correctly with platforms and walls
